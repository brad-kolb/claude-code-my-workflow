# Knowledge Base Wiki Conventions

## Auto-Maintained Files

- `INDEX.md` is generated by `/kb-compile` and `/kb-ingest`. Never edit manually.
- `.manifest.md` is updated by `/kb-ingest`. Never edit manually.

## Article Structure

### Concept articles (`wiki/concepts/`)
Follow `templates/kb-concept.md`: Title, Category, Definition, Key Points, Details, Sources (backlinks), Related Concepts, Notes.

### Source summaries (`wiki/summaries/`)
Follow `templates/kb-summary.md`: Title, Type, Authors, Key Takeaways, Summary, Concepts Extracted, Methodology, BibTeX, Notes.

### Connection articles (`wiki/connections/`)
Cross-cutting themes that span multiple concepts. Include: theme description, connected concepts with links, relationship explanation, implications.

## Linking

- Use standard markdown links: `[text](relative/path.md)`
- No Obsidian-style `[[wikilinks]]`
- Every concept article must link back to its source summaries
- Every summary must link to concepts it contributed to

## Naming

- All filenames: kebab-case (e.g., `difference-in-differences.md`)
- No spaces, no uppercase in filenames

## Preserved Sections

When recompiling articles, preserve any `## Notes` section. This is where human annotations live.

## Integration with Quarto

Wiki outputs in `.qmd` format should follow the project's Quarto CSS conventions (see CLAUDE.md CSS Classes table).

## Source Ingestion

- URLs: use Jina Reader (`https://r.jina.ai/[url]`) via WebFetch for clean markdown extraction
- PDFs: use Read tool (up to 20 pages per call) for papers; `pdftotext` for longer documents
- Local files: copy directly into `raw/`
