# Single Source of Truth: Enforcement Protocol

**The Quarto `.qmd` file is the authoritative source for ALL content.** Everything else is derived.

## The SSOT Chain

```
Quarto .qmd (SOURCE OF TRUTH)
  ├── Rendered HTML / PDF / DOCX (derived)
  ├── Bibliography_base.bib (shared)
  ├── Figures/ (assets — SVG, PNG, R-generated)
  └── scripts/R/ (data source for embedded chunks)

NEVER edit derived artifacts (docs/, rendered output) independently.
ALWAYS edit the .qmd source and re-render.
```

---

## Asset Freshness Protocol

**Before referencing any figure or data file in a QMD, verify it is current.**

1. Check that referenced image files exist in `Figures/` or are generated by R code chunks
2. If using external data (RDS, CSV), verify the file exists and was generated by the current version of the R script
3. If a figure is generated by an R script, verify the script has been run after the last code change

---

## Content Fidelity Checklist

```
[ ] All referenced figures exist on disk
[ ] Math check: every equation renders correctly
[ ] Citation check: every @key resolves in Bibliography_base.bib
[ ] CSS class check: every styled div uses a class defined in theme SCSS
[ ] No broken cross-references or links
[ ] Rendered output matches source (no stale HTML/PDF)
```
